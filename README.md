# demolitions (κατεδαφίσεις)

Εργαλείο έρευνας για τη χαρτογράφηση των αδειών κατεδάφισης στην Ελλάδα
(στο πνεύμα του [«Ο διαρκής θάνατος της μονοκατοικίας»](https://www.kathimerini.gr/investigations/563435530/o-diarkis-thanatos-tis-monokatoikias/)).

Αντλεί από τη **Διαύγεια** τις τελικές «Άδειες Κατεδάφισης» που δημοσιεύει το
ΤΕΕ μέσω του e-Άδειες, κατεβάζει το PDF κάθε άδειας, εξάγει τα στοιχεία
διεύθυνσης και το **περίγραμμα του κτίσματος** (συντεταγμένες ΕΓΣΑ87 από τη
φόρμα, μετατροπή σε WGS84 — η συντριπτική πλειονότητα των αδειών το έχει)
και παράγει spreadsheet. Όπου το PDF δεν δίνει συντεταγμένες, γεωκωδικοποιεί
τη διεύθυνση με Nominatim/OpenStreetMap.

## Εγκατάσταση

Χρειάζονται **Python 3** και το **pdftotext** (πακέτο poppler). Σε
Debian/Ubuntu/Mint όλα μπαίνουν με μία εντολή:

```sh
sudo apt install python3-requests python3-openpyxl python3-flask poppler-utils
```

Εναλλακτικά (macOS, ή όποιος προτιμά pip) με virtualenv — στα σύγχρονα
Linux το σκέτο `pip install` εκτός venv δεν επιτρέπεται:

```sh
brew install poppler                     # macOS· σε Linux: sudo apt install poppler-utils
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt    # requests, openpyxl, flask
```

(Με venv, στις παρακάτω εντολές αντικαταστήστε το `python3` με
`.venv/bin/python3`.)

## Εκτέλεση — Web UI (για μη ειδικούς)

```sh
python3 webui.py
```

Ανοίγει μόνο του τον browser σε μια απλή σελίδα: επιλογή περιοχής με
autocomplete, διάστημα ημερομηνιών, ζωντανή πρόοδος, και στο τέλος κουμπιά
λήψης (Excel, zip με όλα τα PDF) και διαδραστικός πίνακας με ταξινόμηση
και φίλτρα. Με το κουμπί «Χάρτης» τα αποτελέσματα εμφανίζονται ως σημεία
σε χάρτη OpenStreetMap (ομαδοποιημένα, χρωματισμένα κατά ακρίβεια
γεωκωδικοποίησης, με popup ανά άδεια)· τα φίλτρα του πίνακα ισχύουν και
στον χάρτη. Το spreadsheet βγαίνει **αμέσως**· οι συντεταγμένες
προστίθενται μετά ως εμπλουτισμός (μπορεί να ακυρωθεί και να συνεχιστεί
αργότερα με το κουμπί «Προσθήκη συντεταγμένων»). Η καρτέλα «Ιστορικό»
δείχνει όλα τα παλιότερα runs (αποθηκεύονται στο `runs/`).

Τερματισμός: Ctrl-C στο τερματικό. Η σελίδα τρέχει μόνο τοπικά
(127.0.0.1) — δεν είναι προσβάσιμη από το δίκτυο.

## Εκτέλεση — γραμμή εντολών

```sh
python3 -m demolitions --area "Δήμος Δράμας" --from 01/01/2021 --to 31/12/2021 -o drama
python3 -m demolitions --area "Νομός Καβάλας" --from 01/01/2019 -o kavala
python3 -m demolitions --area "Περιφέρεια Κρήτης" --from 01/01/2024 -o kriti
python3 -m demolitions --area "Ελλάδα" --from 01/01/2023 --to 31/12/2023 --no-geocode -o ellada2023
```

Ημερομηνίες σε μορφή ΗΗ/ΜΜ/ΕΕΕΕ (δεκτή και η ΕΕΕΕ-ΜΜ-ΗΗ).

## Δημόσια εγκατάσταση (δωρεάν: Render + Cloudflare R2)

Το app τρέχει σε container (χρειάζεται το `pdftotext`) και κρατά το ιστορικό
των run σε αποθήκη εκτός του (εφήμερου) δίσκου του host.

1. **Cloudflare R2** (δωρεάν 10 GB): φτιάξε ένα bucket και ένα API token
   (Access Key + Secret). Σημείωσε το Account ID.
2. **Render**: New → Blueprint, δείξε στο GitHub repo (διαβάζει το
   `render.yaml`). Στο dashboard όρισε τα μυστικά:
   `R2_BUCKET`, `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`.
   Πάρε το **Deploy Hook URL** του service.
3. **GitHub**: στο repo → Settings → Secrets → Actions, πρόσθεσε
   `RENDER_DEPLOY_HOOK_URL`. Από εκεί και πέρα κάθε push στο `main` τρέχει
   τα tests και, αν περάσουν, αναπτύσσει αυτόματα (βλ. `.github/workflows/ci.yml`).

Στο hosted περιβάλλον τα PDF αποθηκεύονται στο R2 μόνο για τα **πιο πρόσφατα**
run· μόλις το σύνολό τους ξεπεράσει το `DEMOLITIONS_PDF_CACHE_LIMIT`
(default 1 GB) σβήνονται τα PDF των παλαιότερων run (τα μεταδεδομένα μένουν,
και το zip τους ξαναφτιάχνεται κατ' απαίτηση από τη Διαύγεια). Η δωρεάν
βαθμίδα του Render «κοιμάται» μετά από 15′ αδράνειας — η πρώτη επίσκεψη μετά
αργεί ~1′. Ρυθμίσεις μέσω env: `DEMOLITIONS_STORAGE` (`local`|`r2`),
`DEMOLITIONS_PDF_CACHE_LIMIT`, `DEMOLITIONS_CACHE_DIR`.

## Αρχιτεκτονική

- **`demolitions/`** — ο πυρήνας: `diavgeia` (αναζήτηση + ταξινόμηση είδους),
  `pdfparse` (κατέβασμα/ανάλυση φόρμας, εξαγωγή ορόφων/έκτασης/περιγράμματος),
  `egsa87` (ΕΓΣΑ87→WGS84), `geocode` (Nominatim fallback), `areas` (Καλλικράτης),
  `greek`/`dimoi_data` (τονισμός), `output` (xlsx), `pipeline` (ενορχήστρωση με
  callbacks log/step/cancel), `storage` (backends).
- **`webui.py`** — Flask: σερβίρει τη σελίδα, τρέχει **ένα job τη φορά** σε
  background thread, ο browser ρωτά πρόοδο με polling (`/api/status`). Κοινός
  πυρήνας με το CLI (`python3 -m demolitions`).
- **Ροή run (hosted):** αναζήτηση → ανάλυση PDF (staging σε εφήμερο δίσκο) →
  **ανέβασμα στο R2** (τα τοπικά PDF ελευθερώνονται αμέσως μετά) → εμφάνιση
  αποτελεσμάτων → γεωκωδικοποίηση ως εμπλουτισμός (ανεβαίνουν ξανά μόνο τα
  μικρά json/xlsx, όχι τα PDF) → εκκαθάριση cache PDF αν χρειαστεί.
- **Storage:** `LocalStorage` (δίσκος, `runs/<id>/`) ή `R2Storage` (αντικείμενα
  `runs/<id>/<αρχείο>`). Το ίδιο interface· το web σερβίρει/φτιάχνει zip με
  streaming και από τα δύο.
- **Καταγραφή/debug:** stdout/σφάλματα → Render **Logs**· η πρόοδος κάθε run
  φαίνεται ζωντανά στο UI (και στο `/api/status`).
- **Προσοχή (εφήμερος δίσκος):** μια πολύ μεγάλη περιοχή (νομός/όλη η χώρα)
  κατεβάζει εκατοντάδες PDF προσωρινά πριν το ανέβασμα — μπορεί να πιέσει τον
  δίσκο της δωρεάν βαθμίδας. Το όριο cache προστατεύει το R2 μακροπρόθεσμα, όχι
  τον προσωρινό δίσκο· για τέτοιες περιοχές προτιμήστε τοπική εκτέλεση.

## Tests

```sh
python3 tests.py          # offline· χρειάζεται «pip install moto» για το R2 test
```

Κάθε run παράγει έναν φάκελο:

```
drama/
├── drama.xlsx
└── pdf/<δήμος>/<έτος>/<ΑΔΑ>.pdf   # όλα τα PDF των αδειών του run
```

Η στήλη «PDF» του spreadsheet δείχνει στο τοπικό αρχείο (σχετική διαδρομή —
ο φάκελος μεταφέρεται/μοιράζεται αυτούσιος) και η στήλη «ΑΔΑ» στη Διαύγεια.

`--area` δέχεται δήμο, νομό/περιφερειακή ενότητα, περιφέρεια, «Ελλάδα», ή
πολλά χωρισμένα με κόμμα. Σκέτο όνομα νομού (π.χ. «Καβάλας», «Μαγνησίας»,
«Κυκλάδων») ερμηνεύεται ως ο παλιός νομός· για τη στενότερη περιφερειακή
ενότητα γράψτε «ΠΕ Καβάλας». `--no-geocode` παραλείπει τις συντεταγμένες
(πολύ πιο γρήγορο για μεγάλες περιοχές).

## Το spreadsheet

- **Κατεδαφίσεις** — μία γραμμή ανά άδεια: ΑΔΑ (link στη Διαύγεια), ημερομηνία,
  έτος, είδος, έκταση (ολική/τμηματική), δήμος, δημοτική ενότητα/περιοχή,
  πόλη/οικισμός, οδός, αρ. από/έως, ΟΤ, ΚΑΕΚ, περιγραφή έργου, όροφοι
  (1 = ισόγεια/μονώροφη, 2 = διώροφη, …, κενό = δεν προσδιορίζεται),
  συντεταγμένες, ακρίβεια θέσης, στήλη ελέγχου (βλ. Μεθοδολογία).
- **Ανά έτος-δήμο** — pivot με πλήθη ανά έτος και δήμο (μόνο αυτοτελείς
  άδειες κατεδάφισης)· οι «οικοδομικές με κατεδάφιση», αν υπάρχουν, σε
  δικό τους φύλλο.

## Μεθοδολογία

**Πηγή.** Το opendata API της Διαύγειας. Από τον Οκτώβριο 2018 (e-Άδειες)
όλες οι οικοδομικές πράξεις της χώρας δημοσιεύονται από το ΤΕΕ
(organizationUid 99201077, τύπος πράξης 2.4.6.1). Η αναζήτηση γίνεται σε
παράθυρα ≤6 μηνών (όριο του API)· το φιλτράρισμα δήμου γίνεται client-side
με τον κωδικό Καλλικράτη των μεταδεδομένων.

**Τι μετράει.** Μόνο **τελικές** άδειες, μία γραμμή ανά ΑΔΑ, δύο είδη
(στήλη «Είδος»):

- **κατεδάφιση** — αυτοτελής «Άδεια Κατεδάφισης»· αυτές μετρά το pivot
  «Ανά έτος-δήμο».
- **οικοδομική με κατεδάφιση** — ενιαία «Οικοδομική Άδεια» που αναφέρει
  κατεδάφιση στην περιγραφή της (συνήθως κατεδάφιση-και-ανέγερση)· σπάνιες
  (~40 πανελλαδικά από το 2019), σε ξεχωριστό pivot. Οικοδομικές που
  περιλαμβάνουν κατεδάφιση **χωρίς να το γράφουν στο θέμα** δεν
  ανιχνεύονται — μικρή πηγή υποεκτίμησης.

**Τι δεν μετράει.** Προεγκρίσεις, αναθεωρήσεις/ενημερώσεις (θα μετρούσαν
το ίδιο κτίσμα δεύτερη φορά), «Εγκρίσεις Εκτέλεσης Εργασιών» κατεδάφισης
(επικινδύνως ετοιμόρροπα κ.λπ.).

**Καμία σιωπηλή αφαίρεση.** Ό,τι θα μπορούσε να αλλοιώσει το μέτρημα
σημαίνεται σε στήλη και φιλτράρεται από τον χρήστη, δεν αφαιρείται:

- **«Έκταση»** — «ολική» ή «τμηματική/μερική», με συναγωγή από την
  περιγραφή (η φόρμα δεν έχει ρητό πεδίο)· οι τμηματικές δεν είναι
  απώλεια κτίσματος.
- **«Έλεγχος: πιθανό διπλό»** — ίδιο κτίσμα με >1 τελικές άδειες
  (επανεκδόσεις με ίδια διεύθυνση/περιγραφή).
- **«Έλεγχος: ~Xkm από τον δήμο»** — λάθος δηλωμένος δήμος στο e-Άδειες
  (π.χ. άδειες Ηρακλείου Κρήτης στον ομώνυμο δήμο Αττικής).

**Θέση.** Κατά σειρά προτεραιότητας (στήλη «Ακρίβεια συντ/νων»):
1. **κτίσμα (PDF)** — το περίγραμμα του κτίσματος από το πεδίο
   «Συντεταγμένες» της φόρμας (ΕΓΣΑ87 → WGS84, ~98% των αδειών)·
2. αλλιώς Nominatim/OSM κλιμακωτά: οδός+αριθμός → οδός → οικισμός → δήμος.

**Όροφοι.** Από την περιγραφή (ΔΙΩΡΟΦΗ, «2 ΟΡΟΦΩΝ», «ΔΥΟ ΟΡΟΦΩΝ»…)·
κενό = δεν προσδιορίζεται.

**Σύγκριση με την έρευνα της Καθημερινής (Αττική 2019–2024).** Δικό μας
σύνολο 6.220 τελικές άδειες, άρθρου 5.582 (στοιχεία SeDe/ΕΚΚΕ). Η διαφορά
είναι ορισμών, όχι δεδομένων: το άρθρο μετρά μόνο «ολικές» κατεδαφίσεις
κτιρίων κατοικίας/αποθήκης/καταστήματος, **συν**υπολογίζει
αναθεωρήσεις/ανανεώσεις, και εξαιρεί τα νησιά της Αττικής· εμείς μετράμε
όλες τις τελικές άδειες με σημαίες. Οι ακριβείς κανόνες του SeDe δεν
είναι δημοσιευμένοι, οπότε πλήρης ταύτιση δεν είναι εφικτή.

## Περιορισμοί

- Κάλυψη από **Οκτώβριο 2018** (έναρξη e-Άδειες)· παλαιότερες άδειες δεν
  υπάρχουν ομοιόμορφα στη Διαύγεια.
- Για τις λίγες άδειες χωρίς συντεταγμένες στο PDF, η γεωκωδικοποίηση
  εξαρτάται από το OSM· ακίνητα χωρίς οδό/αριθμό πέφτουν στο κέντρο του
  οικισμού ή του δήμου (βλ. στήλη «Ακρίβεια συντ/νων»).
- Το Nominatim έχει όριο ~1 αίτημα/δευτερόλεπτο· για μεγάλες περιοχές η
  πρώτη εκτέλεση αργεί (τα αποτελέσματα κρατιούνται σε cache).

## Cache

Όλα (αναζητήσεις, PDF, γεωκωδικοποιήσεις) αποθηκεύονται στο `cache/`· οι
επανεκτελέσεις είναι σχεδόν ακαριαίες. Σβήστε το `cache/search/` για να
ξαναφέρει πρόσφατα δημοσιευμένες πράξεις σε ήδη κατεβασμένα διαστήματα.
